Accessibility

Testing with Lighthouse

To make sure our website is up to date in terms of accessibility, we chose to test it with Google Chromes integrated program: Lighthouse.

Testing several aspects of our site, also including Performance and Best practices, allows for the best possible result. This was done both for mobile devices and desktop devices, thus making sure we kept both types of users in mind.

This is software we've tested with before and that we feel comfortable using. Both giving us a percentile score as well as more in depth comments to look through. This allows both for a quick feel for how well the page is laid out as well as discerning more specific problems.

All five pages of our website were tested using this program, as previously mentioned both with the desktop and the mobile setting, below you can read a bit about the results these tests provided us with.

Results for mobile and desktop

After running our tests, the results came back pretty posistive! Our colour choices seemed to be strong, contrast and visibility were good and the site is friendly for people struggling with colour perception.

In addition test gave us 100% for best practice accross the board, both for correct use of elements as well as remembering alt tags for pictures and similar semantics.

One thing that did pop up as an issue was the performance for mobile, this was largely due to images being served as png or jpg, rather than "next-gen format" with better compression such as WebP and AVIF. In addition to this a couple of percentages was subtracted due to the sizing of images, however these images are supposed to be responsive based on the size of the viewport, and we're therefore happy with their current size.

Other than that, we passed audits for things unused css, having correct meta tags, having valid "lang" attribute and having a title. Headings appear in sequentially descending order and all list items are contained within valid parent elements. The only issue that came up regarding accessibility was the fact that there is a "p"-tag inside an "ul", but since we don't use any javascript this shouldn't cause any problems.

Scalability and navigation

We have emphasized developing a website that is responsive to size changes.

The most noticable feature that helps achieving this is that the elements stacks vertically instead of horizontally when the viewport is below 800px from left to right. The picture to the left is a screenshot of the CSS-snippet that makes this happen.

Another thing that happens simultaneously is that the menu turns into a hamburger-menu. This allows phone-users to navigate seamlessly between the pages.

We have also added a "back to the top"-button that makes it very easy and quick to reach the top of the page instantly. As some of the pages has a decent amount of content, and hence needs a bit of scrolling to go up and down, this feature should notably improve the user experience.
